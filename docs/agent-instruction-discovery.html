01 / Taxonomy

Four ways to build context.

The filename is the least interesting part. The decisive behavior is whether an agent traverses ancestors, activates rules after touching a file, applies metadata, or waits for explicit input.

A / DYNAMIC

Chain + lazy descendants

Load the broad chain now. Add local instructions when a tool enters a subtree.

Claude · Gemini · Amp · goose · OpenCode

B / STATIC

Fixed ancestor chain

Compose root to working directory once. Descendants below launch CWD remain invisible.

Codex · pi · Qwen*

C / REACTIVE

Conditional rule engine

Use globs, descriptions, modes, model judgment, or manual mentions to activate rules.

Cursor · Windsurf · Cline · Roo · Continue

D / EXPLICIT

Root or opt-in context

Read a fixed root scope, or require the user to attach persistent context directly.

Junie · Kiro · Aider

04 / Fault lines

The differences that break assumptions.

Repositories often carry several instruction files because compatibility is asymmetric. Even shared filenames can imply different traversal and precedence.

↥

The same CWD has different parents.

Claude and pi may reach the filesystem root. Amp reaches home. Codex and OpenCode stop at project boundaries. Root detection is harness behavior, not generic setup.

≠

Override does not mean override.

Codex uses replacement to choose one file in a directory. Cursor defines tier precedence. Most other agents merely order contradictory prose and ask the model to reconcile it.

⌁

There may be no final context.

Lazy subtree and glob rules mean context evolves as the agent reads files. A useful trace must show initial context and every later activation event.

@

Imports are dialects, not a standard.

Depth ranges from three to five where published. Some allow globs, some reject paths outside the repo, some ignore code blocks, and OpenCode deliberately does not parse them.

05 / Build notes

What the CLI must remember.

A per-harness evaluator can share filesystem primitives, but it must preserve why each instruction appeared, what it displaced, and how certain that behavior is.

Provenance

  • Path or remote URL
  • Managed, global, project, directory, local, or session scope
  • Ancestor, fallback, import, glob, or explicit discovery reason
  • Documentation or source-code evidence

State

  • Selected, shadowed, excluded, empty, truncated, or over budget
  • Always, startup, path, model, manual, task, or request activation
  • Known prompt position
  • Unresolved semantic precedence

Simulation

  • Harness version and configuration
  • Launch directory and detected boundary
  • Initial context snapshot
  • Context after one or more accessed paths
